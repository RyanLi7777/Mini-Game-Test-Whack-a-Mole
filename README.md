# 欢乐打地鼠 —— 本地运行 / 测试说明

> 本项目严格依据《欢乐打地鼠-需求文档.md》（v1.1，唯一权威需求来源）实现，完全离线、配置驱动、素材热替换。

## 一、如何运行（两种方式任选）

### 方式 A：双击直接打开（最简单，推荐没有命令行经验的用户）

直接双击 **`index.offline.html`** 用浏览器打开即可玩，**不需要安装任何东西、不需要命令行**。

> 原理：普通 `index.html` 用的是 ES Module（`<script type="module">`），浏览器对 `file://` 协议下的模块有跨域限制，双击打不开。`index.offline.html` 引用的是预先打包好的 `dist/bundle.js`（普通脚本，无模块限制），所以可以直接双击运行。

⚠ **如果你修改了 `src/` 或 `config/` 下的源码**，`index.offline.html` 不会自动感知变化，需要重新打包：
```bash
node tools/build.mjs
```
（仅替换 `images/`、`audio/` 里的素材文件**不需要**重新打包，刷新页面即生效。）

### 方式 B：本地服务器（适合需要改代码、调试的开发场景）

`index.html`（ES Module 版）必须用本地静态服务器打开，本项目已附带一个零依赖的 Node 服务器：

```bash
node tools/serve.mjs 8000
```

然后浏览器打开：
- 游戏：`http://localhost:8000/index.html`
- 单元测试：`http://localhost:8000/tests.html`

> 若你本机的 `python`/`npx serve` 可用，也可以用它们代替（`python -m http.server 8000`），效果相同。

> 两种方式启动后均可断网游玩（首次加载后无任何网络请求、无外部 CDN/字体）。

### ⚠ 重要修复记录
此前 `src/assets.js` 中图片/音频请求路径多写了一层 `assets/` 前缀（如 `assets/images/...`），
而项目实际目录结构里 `images/`、`audio/` 与 `index.html` 同级，导致**所有真实素材文件请求一直 404**，
游戏此前显示的动物/锤子其实是程序生成的占位圆圈，并非你放入的真实图片，音频也完全没有播放。
现已修正为 `images/`、`audio/`（已通过 HTTP 200 验证），真实素材现在会被正确加载。

## 五、交付说明 / 额外假设（文档未明确处，按行业稳妥默认）

1. **技术栈**：原生 HTML5 + Canvas 2D，零运行时依赖，完全离线；提供 ES Module（开发）与打包后单文件（双击直接打开）两种交付形态。
2. **「新游戏」入口**：选择「进入选关界面并定位到第 1 关」（§3.2 B 二选一）。
3. **占位资源**：缺失图片采用**运行时程序绘制**的风格统一占位，缺失音频回退占位 `.wav`；放入正式文件后自动覆盖，无需改代码（满足「只换文件」）。
4. **地鼠外观数量**：由配置 `assets.moleSkinCount` 控制（默认 3）；新增 `mole_4.png` 时把该值改为 4 即生效。
5. **重置/新游戏清空**：清空 `levels` 与 `unlockedLevel`，**保留音量设置**（§8.4 允许）。
6. **暂停**：暂停时音乐暂停（§7.1 二选一中取「暂停」）。
7. **每关 `boardArea`**：按 3×3/4×4/5×5 给了居中网格区域（背景不含洞口，无需对齐，可随时微调）。
8. **素材抠透明**：用户提供的动物/锤子/洞口等 PNG 原本不带 Alpha 通道（纯色背景），已用 `tools/make-transparent.mjs` 做颜色距离抠图 + 边缘羽化处理为透明背景，原图备份在 `images_backup/`。若效果不理想（误抠/毛边），建议用图像工具精修后替换同名文件。
8. **音频自动播放**：浏览器策略可能拦截首次 BGM，已在首次用户手势后自动重试播放。
