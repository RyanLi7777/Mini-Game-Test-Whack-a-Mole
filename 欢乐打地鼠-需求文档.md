1. **配置驱动**：游戏的几乎所有数值（关卡时长、出怪节奏、概率、计分、星级阈值、棋盘几何等）都必须集中在一份配置文件里（见 §6），代码读取配置运行。**严禁把数值硬编码进逻辑**。目标是：策划/玩家日后只改配置文件即可调整难度与手感，无需改动代码。
2. **完全离线**：成品必须在断网环境下完整可玩。**不得依赖任何外部 CDN、在线字体、远程接口**。所有依赖、字体、库都本地打包。

5. **代码结构**：模块化、可读、含必要注释；逻辑分辨率与计分等核心逻辑要便于单元测试。

## 1. 项目概述

| 项 | 内容 |
|---|---|
| 游戏名称 | 欢乐打地鼠 |
| 类型 | 单机休闲 / 打地鼠（关卡制） |
| 形态 | HTML5 网页游戏，完全离线可玩 |
| 平台 | PC 浏览器 + 手机浏览器（一套代码） |

## 2. 技术与平台要求

### 2.1 技术形态
- HTML5 单机网页游戏，浏览器直接打开即玩。
- 完全离线运行，无任何网络请求。
- 推荐使用 Canvas 2D 进行渲染（精灵、动画、命中检测可控且性能稳定）；如采用 DOM/CSS 方案需保证手机端 60fps。具体技术栈由实现者选择，但必须满足上述离线与跨端约束，且所有依赖本地化。

### 2.2 分辨率与自适应
- 以 **1920×1080** 为逻辑坐标系；所有布局坐标、棋盘几何、HUD 位置都基于此坐标系定义。
- 实际渲染采用 **letterbox（等比缩放 + 留黑边）**，保证任何屏幕比例下画面不变形、内容完整可见。
- 横屏布局。手机端**竖持时**显示"请横屏游玩"全屏遮罩，转回横屏自动隐藏并继续。
- 逻辑分辨率写入配置（§6.1 `resolution`），便于日后整体调整。

### 2.3 性能与兼容
- 目标 60fps，需在中端手机浏览器流畅运行。
- 支持主流现代浏览器最新版：Chrome、Safari（含 iOS Safari）、Firefox、Edge。
- 资源（图片/音频）需做预加载与加载进度反馈，避免首次进入卡顿或无图。

### 2.4 输入
- PC：鼠标移动控制光标，鼠标左键点击 = 挥锤。
- 手机：触摸点击/点按 = 挥锤（点按位置即落锤位置）。
- 同时支持鼠标与触摸，互不冲突。

## 4. 核心玩法规则

### 4.1 棋盘（洞口布局）

| 关卡 | 棋盘尺寸 | 洞口数 |
|---|---|---|
| 1–3 | 3×3 | 9 |
| 4–7 | 4×4 | 16 |
| 8–9 | 5×5 | 25 |

- 每个洞口是一个出怪位（cell）。同一洞口同一时刻只容纳一只动物。
- **洞口位置由配置定义、由程序绘制**：代码依据棋盘像素区域 + 行列数（见 §6.3 `boardArea`/`boardRows`/`boardCols`）计算每个洞口中心坐标，并在该位置绘制洞图 `hole.png`（见 §5、§7.2）。**背景图仅作场景、不含洞口**，因此 `boardArea` 自由定义网格布局即可，无需与背景对齐；只需保证棋盘区域在背景上视觉协调（可微调 `boardArea`）。

### 4.2 三种动物

| 动物 | 出现关卡 | 形象数量 | 敲击效果 | 对连击的影响 |
|---|---|---|---|---|
| 地鼠 | 1–9 关 | **3 种外观**（机制完全相同，随机出现） | 基础得分 + 连击加成 | **推进连击**（占连击位） |
| 老鼠 | 4–9 关 | 1 种 | 固定分（不翻倍）+ 触发/刷新限时双倍 | **不占连击位、不中断连击** |
| 猫 | 4–9 关 | 1 种 | 扣分（不翻倍）、分数最低到 0 | **中断连击**（清零） |

- 前 3 关只有地鼠；第 4 关起加入老鼠和猫。
- 三种地鼠外观仅视觉不同，分值/速度/行为完全一致。

### 4.3 动物生命周期
- **冒出**：从洞中升起（上移 + 轻微挤压动效），约 0.15s。
- **停留**：保持可见，时长 = 该关 `moleStayTime`（见 §6.3，所有动物共用同一停留时长）。
- **自行缩回**：未被敲击则到时缩回（下沉动效），约 0.15s。**自行缩回不影响连击、不计分**。
- **被敲击**：见 §4.5、§7。

### 4.4 锤子与光标
- **仅在关卡进行中、且光标位于游戏区（棋盘区域）内时**，光标显示为锤子图，可向下砸；其余界面/区域光标为手指状（手指光标图由用户提供）。
- PC 端锤子跟随鼠标移动；点击触发"向下砸"动作（落锤动画，约 0.1s 落下回弹）。
- 手机端点按即在点按处落锤。
- 锤子的"常态/抬起"与"砸下"通过素材 + 代码动效实现（见 §5、§7）。

### 4.5 命中判定
- 一次挥锤（点击/点按）对落点做命中检测。
- **单次挥锤若同时压住多只动物 → 全部命中**（见 §4.7 多重命中结算顺序）。
- **空砸**：在游戏区内点到非动物处 → 算空砸，**中断连击**（连击清零），播放挥锤音效。
- 点击顶部 HUD / 按钮等 UI 元素**不算挥锤**，不触发落锤、不影响连击。

### 4.6 计分与连击（核心规则）

**基础计分**
- 地鼠基础分：每只 +1。
- 连击加成（按"连击位次"计，仅地鼠占位）：
  - 第 1 只：+1（即基础分）
  - 第 2–5 只：每只 **+2**
  - 第 6 只及以上：每只 **+3**
- 老鼠：固定分 `ratScore`（默认 +1，可调），**不翻倍**，**不改变连击位次**。
- 猫：扣 `catPenalty`（默认 2），**不翻倍**，分数 **最低到 0、不为负**（`scoreFloor = 0`）。

**连击规则**
- "连击位次"`combo` 只统计**连续敲中的地鼠数**。
- **中断连击的唯一两种情形**：① 空砸（游戏区内挥锤未命中任何动物）；② 敲中猫。
- **不中断连击**：动物自行缩回、玩家未挥锤、敲中老鼠，均**不**中断。
- 连击**每关清零**，不跨关保留；每关开始时 `combo = 0`。

**限时双倍（老鼠 Buff）**
- 敲中老鼠 → 激活"双倍"，持续 `doubleBuffDuration`（默认 5s）。
- 双倍生效期间再次敲中老鼠 → **刷新**为满 5s（不叠加时长，重置计时）。
- 双倍**仅作用于"地鼠基础分 + 连击加成"**：地鼠每次得分 ×`doubleBuffMultiplier`（默认 ×2）。
- 双倍**不**作用于：老鼠固定分、猫的扣分。
- 双倍与连击加成**叠加**。例：双倍期内第 6 只连击地鼠 = 基础计分 3 × 2 = **6 分**。

### 4.7 多重命中结算顺序（消除歧义）
当**一次挥锤同时命中多只动物**时，按**固定顺序**结算，保证结果确定、与处理次序无关：
1. **先结算所有地鼠**：依次推进 `combo` 并按位次计分（受双倍影响）。
2. **再结算所有老鼠**：各加 `ratScore`（不翻倍），激活/刷新双倍。
3. **最后结算所有猫**：各扣 `catPenalty`（不翻倍，分数不低于 0），随后 `combo` 清零。

> 含义：若同一锤同时敲到地鼠与猫，玩家仍获得该地鼠的（含连击的）得分，但随后猫使连击清零并扣分。

### 4.8 计分逻辑伪代码（实现参考）

```text
# 配置见 §6
function onSwing(pointerPos):
    play(SFX.hammer_swing)
    animateHammerDown(at = pointerPos)

    contacted = animalsUnder(pointerPos)   # 可能为 0 / 1 / 多只

    if contacted is empty:
        combo = 0                          # 空砸：中断连击
        return

    moles = contacted.filter(MOLE)
    rats  = contacted.filter(RAT)
    cats  = contacted.filter(CAT)

    # 1) 地鼠：推进连击并计分（按出现先后）
    for mole in moles:
        combo += 1
        pts = comboPointsFor(combo)        # 1→1; 2..5→2; >=6→3 （读配置）
        if doubleBuffActive: pts *= config.doubleBuffMultiplier   # 仅地鼠
        score += pts
        floater(mole.pos, "+" + pts)
        playHitAnim(mole)                  # 切换被砸表情 → 挤压下沉
    if moles not empty: play(SFX.hit_mole)

    # 2) 老鼠：固定分、不改连击、刷新双倍
    for rat in rats:
        score += config.ratScore           # 不翻倍
        activateOrRefreshDoubleBuff(config.doubleBuffDuration)
        floater(rat.pos, "×2!")
        playHitAnim(rat)
    if rats not empty: play(SFX.hit_rat)

    # 3) 猫：扣分、最低 0、清空连击（最后结算）
    for cat in cats:
        score = max(config.scoreFloor, score - config.catPenalty)  # 不翻倍
        floater(cat.pos, "-" + config.catPenalty)
        playHitAnim(cat)
    if cats not empty:
        combo = 0
        play(SFX.hit_cat)

function onRetreat(animal):                # 自行缩回
    animateSink(animal)                     # 不计分、不影响连击
```

### 4.9 出怪逻辑（实现参考）
- 每隔 `spawnInterval × (1 ± spawnJitter)` 秒尝试生成一只动物。
- 仅当当前在场动物数 < `maxConcurrent` 时生成。
- 从空闲洞口中随机选择一个。
- 按该关 `spawnWeights`（地鼠/老鼠/猫权重）加权随机决定类型；**1–3 关恒为地鼠**。
- 若为地鼠，再从 3 种外观中随机其一。
- 新动物停留 `moleStayTime` 秒后自行缩回。

### 5.2 命名规则
- 全部小写，单词用下划线连接，扩展名固定（图片 `.png`、音频 `.mp3`）。
- 关卡相关素材以序号结尾：`level_1` … `level_9`、`mole_1`…`mole_3`。
- "被砸表情"以 `_hit` 结尾。
- 用户后续新增素材（如更多地鼠外观 `mole_4.png`），数量上限由配置 `assets.moleSkinCount` 之类的项控制，或由代码自动探测连续序号——实现者择一并在交付说明注明，确保"加文件即生效"。

### 5.3 美术规格
- 所有动物、锤子、光标为**透明背景 PNG**。
- 以 1920×1080 逻辑分辨率为基准设计，建议为高分屏提供 2x 资源或确保素材足够清晰（实现者可在配置中定义每类素材的显示尺寸）。
- **被砸动画**：用户提供"被砸表情"静态图，**命中时由代码完成过渡**（基础形象 → 切换为被砸表情 → 挤压/下沉 → 退场），过渡要自然、不突兀（见 §7.2）。
- **洞口与背景**：背景图（`level_N.png`）仅作场景、**不含洞口**；洞口由程序在每个 cell 位置绘制 `hole.png`（全关通用，透明背景 PNG）。洞图的显示尺寸由配置定义（可按 cell 大小自适应或单独指定）。如另提供 `hole_front.png`（洞口前沿），程序会将其画在动物之上，实现"从洞后冒出"的遮挡效果。

### 5.4 缺失素材的降级策略（关键，保证可运行可测试）
游戏在**任何素材缺失时都不得报错或白屏**，按以下回退链处理：
- **被砸表情图缺失**：回退链 `mole_N_hit → mole_hit → 对基础图做代码挤压动效`；老鼠/猫同理回退到代码挤压。
- **锤子砸下图 `hammer_down.png` 缺失**：用代码对 `hammer.png` 做旋转/位移模拟砸下。
- **洞图 `hole.png` 缺失**：用代码绘制简单洞口形状（如深色椭圆）兜底；`hole_front.png` 缺失则跳过前沿遮挡、动物做简单升起。
- **关卡背景缺失**：使用 `menu_bg.png` 或纯色背景兜底。
- **任意音频缺失**：静音处理（不播放、不报错）。
- **首次开发时若某些素材尚未提供**：Claude Code 应生成**风格尽量统一的占位图/占位音**（占位音可为极短提示音或留空），确保游戏可立即运行与测试；这些占位资源会在用户放入正式文件后被覆盖。

## 8. 数据持久化（单存档）

### 8.1 存储内容（localStorage）
- 每关已获得的**最高星数**（0–3）与**最高分**。
- 已解锁到的最高关卡（`unlockedLevel`）。
- 设置项：音乐音量、音效音量。
- 存档结构需带版本号字段，便于日后迁移。

存储结构示例（key 如 `happy_whack_save_v1`）：
```json
{
  "version": 1,
  "unlockedLevel": 3,
  "levels": {
    "1": { "stars": 3, "bestScore": 142 },
    "2": { "stars": 2, "bestScore": 121 },
    "3": { "stars": 0, "bestScore": 0 }
  },
  "settings": { "musicVolume": 0.7, "sfxVolume": 0.8 }
}
```

### 8.2 解锁规则
- 初始仅第 1 关解锁。
- 某关达成 ≥1 星 → 解锁下一关（`unlockedLevel` 取已解锁最大值）。
- **重玩已通关关卡**：成绩取历史最高——`stars = max(旧, 新)`，`bestScore = max(旧, 新)`；不会因重玩成绩变差而降级。
- 测试按钮"解锁全部关卡"：直接将全部关卡置为解锁（不改变已有星数）。

### 8.3 局内退出/失败的记录
- 中途**退出**或时间到但**未达 1 星**：不写入更高星数（保持历史最高），按未过关处理。
- 仅当本局成绩高于历史时才更新存档。

### 8.4 重置
- 设置中的"重置进度"与新游戏的"清空确认"，都清空 `levels` 与 `unlockedLevel`（回到仅第 1 关解锁），音量设置可保留（实现者注明）。

## 9. 边界情况与防错

1. 时间到的瞬间正在挥锤/动物正在被砸：本局立即结束，正在结算的命中按已计入处理，随后弹结算。
2. 暂停期间：计时、出怪、动画、Buff 倒计时全部冻结，恢复后继续。
3. 同一洞口不可同时存在两只动物；洞口全满时本次生成跳过。
4. 双倍 Buff 在关卡结束/退出时清除，不跨关。
5. 连续快速点击：每次点击独立判定挥锤；命中已在退场动画中的动物不重复计分。
6. 分数永不为负（下限 `scoreFloor`）。
7. 切后台/失焦（页面隐藏）：建议自动暂停，避免离开时计时空跑。
8. 旋转屏幕：横↔竖切换时正确显示/隐藏"请横屏"遮罩并保持游戏状态。
9. 任何素材/音频缺失：按 §5.4 降级，绝不白屏或报错。
10. localStorage 不可用（隐私模式等）：进度无法保存时降级为单次会话可玩，并可给轻提示（不阻断游戏）。

## 10. 验收标准（Acceptance Criteria）

> 以下为可逐条勾选验证的验收清单。**每一条都应可被独立测试通过。** 标有【数值用例】的项给出了确定的输入与期望输出，须严格通过。

### 10.1 平台与启动
- [ ] 在 PC 浏览器（Chrome/Firefox/Edge/Safari 最新版）打开可正常进入并游玩。
- [ ] 在手机浏览器（iOS Safari / Android Chrome）打开可正常进入并游玩。
- [ ] **断网状态**下完整可玩：刷新后仍可运行，无任何网络请求/外部 CDN 依赖。
- [ ] 任意屏幕比例下采用 letterbox，画面不变形、内容完整、无错位。
- [ ] 手机竖持时显示"请横屏游玩"遮罩，转横屏后自动隐藏并继续。
- [ ] 资源预加载有进度反馈，加载完成进入主菜单，无白图/闪烁。

### 10.2 主菜单与流程
- [ ] 主菜单含"新游戏 / 继续游戏 / 设置"三个按钮且均可用。
- [ ] 首次（无存档）："继续游戏"行为符合 §3.2(B)；"新游戏"直接开始且不弹清空确认。
- [ ] 有存档时点"新游戏"弹二次确认；确认后清空进度并从第 1 关开始；取消则不改动。
- [ ] "继续游戏"进入选关界面并停在已解锁进度。
- [ ] 菜单/设置界面播放菜单 BGM。

### 10.3 选关界面与解锁
- [ ] 列出全部 9 关，每关正确显示锁定状态与已获星数（0–3）。
- [ ] 未解锁关卡不可进入；已解锁关卡可进入。
- [ ] 已通关关卡可重进刷分；刷出更高成绩后星数/最高分更新，刷出更低成绩不降级。
- [ ] 达成某关 ≥1 星后，下一关变为解锁。
- [ ] 【测试】"解锁全部关卡"按钮可一键解锁全部 9 关；将 `debug.showUnlockAllButton=false` 后该按钮不显示。

### 10.4 棋盘与关卡
- [ ] 1–3 关为 3×3、4–7 关为 4×4、8–9 关为 5×5。
- [ ] 每关为限时单局，时长读取配置（默认 60s），HUD 倒计时正确，归零即结算。
- [ ] 动物从洞口冒出、停留、未敲则自行缩回；自行缩回**不计分、不中断连击**。
- [ ] 难度逐关递增可感知：出怪更快、停留更短、出现更频繁、靠后关猫更多（同屏上限遵守 §6.3）。
- [ ] 同屏在场动物数不超过该关 `maxConcurrent`；同一洞口不同时出现两只。

### 10.5 计分与连击（核心）
- [ ] 敲中地鼠：基础 +1；连击第 2–5 只每只 +2；第 6 只及以上每只 +3。
- [ ] 连击仅由"空砸"或"敲中猫"中断；老鼠不中断、自行缩回不中断、未挥锤不中断。
- [ ] 连击每关清零、不跨关。
- [ ] 老鼠：加固定分（`ratScore`），不占连击位、不中断连击，并触发/刷新双倍。
- [ ] 猫：扣 `catPenalty`，中断连击，分数不为负。
- [ ] 单次挥锤同时命中多只 → 全部命中，并按 §4.7 顺序结算（地鼠→老鼠→猫）。
- [ ] 双倍仅作用于地鼠（基础+连击），不放大老鼠固定分与猫扣分；与连击叠加。
- [ ] 双倍持续 5s，期内再敲老鼠刷新为满 5s。
- [ ] 命中/扣分有飘字反馈，双倍有"×2+倒计时"提示。

**【数值用例】（在无其它干扰、连击从 0 起算的前提下逐一验证）**

- [ ] **用例 A（纯连击）**：连续敲中 7 只地鼠（无空砸、无老鼠/猫、无双倍）。期望总分 = 1+2+2+2+2+3+3 = **15**。
- [ ] **用例 B（空砸断连击）**：敲 3 只地鼠（1+2+2=5）→ 空砸（连击清零）→ 再敲 2 只地鼠（1+2=3）。期望总分 = **8**。
- [ ] **用例 C（老鼠不断连击 + 触发双倍）**：敲 2 只地鼠（1+2=3，连击=2）→ 敲老鼠（+1，连击仍=2，双倍生效）→ 在双倍内敲 1 只地鼠（连击=3 → 基础 2 × 2 = 4）。期望总分 = 3+1+4 = **8**。
- [ ] **用例 D（猫扣分 + 断连击 + 下限 0）**：起始 0 → 敲 1 只地鼠（+1，连击=1）→ 敲猫（1−2，下限取 0，连击清零）→ 敲 1 只地鼠（+1，连击=1）。期望总分 = **1**。
- [ ] **用例 E（双倍不放大老鼠/猫）**：双倍生效期间敲老鼠仅 +`ratScore`（不翻倍）；敲猫仅 −`catPenalty`（不翻倍）。
- [ ] **用例 F（双倍刷新）**：敲老鼠后第 4 秒再敲老鼠，双倍重置为满 5s（而非叠加为更长）。
- [ ] **用例 G（一锤多中的结算顺序）**：起始 0，一锤同时命中 1 地鼠 + 1 猫 → 先记地鼠（连击=1，+1）再结算猫（1−2 取下限 0、连击清零）。期望该锤后总分 = **0**，连击 = 0。

### 10.6 三星评分
- [ ] 过关需 ≥1 星；星级按 `targetScore × {0.40, 0.65, 0.85}` 判定。
- [ ] 修改配置中的 `targetScore` 或阈值比例后，星级判定随之变化（验证确为配置驱动）。
